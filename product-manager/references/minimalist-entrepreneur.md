## 1. 十步创业路线图

极简主义创业的核心进展路径：

```
社区 → 验证 → 流程化 → 产品化 → 销售 → 定价 → 营销 → 增长 → 文化 → 复盘

每一步都建立在前一步的基础上。
跳步是大多数创业者失败的原因。
```

**与 Vibe Coding 的结合**：AI 让"产品化"这一步变得极快，但这反而让前面几步（社区、验证、流程化）更加重要——因为你用 AI 做错东西的速度也快了 10 倍。

## 2. 找到你的社区（Find Community）

### 核心原则

**从社区开始，不是从产品开始。** 最好的极简创业来自你已经深入参与的社区——你注意到了一个值得解决的问题。你不需要"寻找"社区，你已经属于很多个了。

### 自问清单

1. **你已经属于哪些社区？** 想宽泛一点：职业群组、兴趣社区、论坛、校友网络、本地组织、家长群、宗教团体……
2. **你在网上哪里花时间最多？** Reddit、Discord、Slack 群、Twitter/X、微信群、知乎、B 站、小红书……
3. **你反复听到人们抱怨什么问题？** 最好的创业想法来自社区中持续存在的、反复出现的痛点。
4. **你愿意为哪个社区服务好几年？** 这不是周末项目——你会长期服务这些人。

### 评估标准

| 维度 | 评估问题 |
|------|---------|
| **真实成员** | 你真的是这个社区的一份子吗？理解他们的语言、价值观、文化？ |
| **付费意愿** | 这个问题痛到人们愿意花钱解决吗？ |
| **可触达** | 你知道他们在哪里聚集？能直接联系到他们吗？ |
| **规模合适** | 足够大到有商业价值，但不大到你永远无法脱颖而出？ |

### 反模式

- ❌ 试图从零创建一个社区，而不是加入已有的社区
- ❌ 纯粹因为市场大而选择社区，而不是因为真正感兴趣
- ❌ 跳过社区参与，直接想"我能卖什么"
- ❌ 目标太宽泛："所有上网的人"

## 3. 验证想法（Validate Idea）

### 核心原则

**验证是通过"卖"来完成的，不是通过"做"。** 大多数创始人花几个月做了一个没人要的产品。正确做法：先手动卖解决方案，再做产品。

### 验证四步法

**第一步：定义问题（不是解决方案）**
- 谁有这个问题？（具体——不是"企业"，而是"月收入 5-10 万的独立设计师，苦于发票管理"）
- 他们现在怎么解决？（现有的变通方案就是你的真正竞争对手）
- 这个问题有多痛？（轻微不便 vs 火烧眉毛）
- 他们愿意付钱解决吗？

**第二步：你能手动解决吗？**
- 在写任何代码之前，你能用手工为他们解决这个问题吗？
- 先雇佣自己。把每一步写在纸上。
- 如果你能手动为几个人解决，你最终就能自动化它。

**第三步：有人愿意付钱吗？**
- 你能现在就为这个手动服务收费吗？
- 你跟至少 10 个潜在客户聊过了吗？
- 至少有 3 个人说愿意付费（或者已经付了）？

**第四步：四个建造前必答题**
1. 我能在一个周末做出来吗？ → 如果不能，缩小范围
2. 它能让客户的生活好一点吗？ → 这就是 MVP 的标准
3. 客户愿意为此付钱吗？ → 从第一天就盈利
4. 我能快速获得反馈吗？ → 反馈循环越快，产品越快变好

### 信号判断

| 绿灯（值得做） | 红灯（不要做） |
|--------------|--------------|
| 人们已经在为劣质解决方案付钱 | 没人在试图解决这个问题 |
| 你手动帮几个人解决了，他们很开心 | 你说不出 10 个有这个问题的人 |
| 社区在积极抱怨这个问题 | 唯一的验证是"朋友觉得这主意不错" |
| 你能一句话描述客户和他们的痛点 | 你需要"教育"人们他们有这个问题 |
| 你在解决自己的痒（scratching your own itch） | 你为一个你不属于的社区做产品 |

## 4. 流程化（Processize）

### 核心原则

**先流程化，再产品化。** 每一个伟大的产品都始于某个人用手工干活。Gumroad 最初是 Sahil 手动收集 PayPal 邮箱，一个一个给创作者付款。你的产品也应该这样开始。

### 六步流程化

**Step 1：描述你的产品想法**
- 这个产品为客户做的**一件事**是什么？
- 客户使用前有什么？使用后有什么？
- 如果完全不用技术——只有你、手机和表格——你会怎么做？

**Step 2：找到今天就需要这个的人**
- 谁已经在用变通方案解决这个问题？
- 你能说出 10 个具体的人吗？

**Step 3：设计手动版本**
- 客户给你什么？（邮件、文件、需求描述）
- 你拿到后做什么？（每一步，按顺序）
- 你给客户什么？（交付物）
- 每个客户要花你多长时间？

**Step 4：写"魔法纸条"**

```
把你的手动流程写在一张纸上，清楚到任何人都能照着做。
如果你去度假，别人能拿着这张纸条接手你的业务。
这就是你的"魔法纸条"。

包含：
1. 触发条件 — 什么启动这个流程？
2. 步骤 — 编号的动作列表
3. 工具 — 每步用什么（表格、邮件、手机）
4. 时间 — 每步耗时
5. 交付 — 怎么把结果给客户
```

**Step 5：收费**
- 免费和 1 块钱之间有巨大的鸿沟（"零价格效应"）
- 这周联系 3 个社区里的人，交付手动版本，收取费用
- 每次互动都教会你一些纯靠编程永远学不到的东西

**Step 6：何时产品化**
- 你已经手动交付给 10+ 付费客户
- 你能准确预测他们会要什么
- 魔法纸条的步骤几周没有变化了
- 你花在交付上的时间比找客户还多
- **这时候——也只有这时候——才开始用 AI / Vibe Coding 自动化**
- 先自动化耗时最多的那一步

### 与 Vibe Coding 的结合

```
传统路径：想法 → 手动编程 6 个月 → 发现没人要 → 死
Vibe Coding 路径：想法 → AI 编程 3 天 → 发现没人要 → 死得更快
极简主义路径：想法 → 手动交付 2 周 → 验证有人要 → AI 编程 3 天 → 活着赚钱
```

## 5. 最小可行产品（MVP）

### 核心原则

**尽可能少地构建。** 目标是尽快开始为社区交付价值。不是做出美丽、精致、完整的东西。

### 三个阶段

```
Manual（手动做）→ Processized（流程化）→ Productized（产品化）

互联网上大多数应用不过是"表单和列表"（CRUD）。
你的 MVP 不应该比这更复杂。
```

### 建造前四问

| 问题 | 如果答案是"否" |
|------|--------------|
| 我能一个周末做出来吗？ | 缩小范围，直到可以 |
| 它让客户的生活好了一点吗？ | 这是 MVP 的最低标准 |
| 客户愿意为此付钱吗？ | 从 Day 1 就盈利 |
| 我能快速获得反馈吗？ | 为能给你反馈的人做 |

### MVP 原则

- **一件事**——你的产品最初只做一件事
- **不要打磨**——不需要好看。Craigslist 从来不好看
- **收费**——免费和 $1 之间有天壤之别
- **用现成工具**——Gumroad、Stripe、Airtable、Google Forms、Zapier、Notion……什么最快上市就用什么

### 不要做的事

- ❌ 做"将来可能需要"的功能
- ❌ 为规模做架构——你还没有规模问题
- ❌ 网站够用时做 App
- ❌ 表格够用时写代码
- ❌ 无代码工具够用时找工程师

### MVP 上线清单

- [ ] 给业务取名（两个真实单词组合 > 生造词；过"电台测试"——听到就能拼出来）
- [ ] 买域名（~$10/年）
- [ ] 做简单网站（Carrd、Gumroad、或类似工具）
- [ ] 创建社交媒体账号（个人 + 业务）
- [ ] 配置支付（Stripe：2.9% + $0.30/笔）
- [ ] 创建客户沟通邮箱

## 6. 前 100 个客户（First Customers）

### 核心原则

**别搞发布。专注于卖。** "病毒式成功"是神话。每一个看似一夜成功的背后，都是数月甚至数年的努力。你的工作是一个接一个地卖，从每次互动中学习。

### 同心圆销售法

从最在乎你的人向外卖：

**第一圈：亲朋好友**
- 从这里开始。是的，很尴尬。做就对了。
- 他们比任何人都更信任你。如果他们不买，谁会？
- 要真诚的反馈，不是社交媒体点赞。

**第二圈：你的社区**
- 你已经贡献过的那个社区里的人
- 三步走：
  1. 列出所有写过/分享过类似东西的人
  2. 逐一私聊联系——带他们看产品，请吃饭，做几百次
  3. 要直接反馈——不是好评，是真实反馈

**第三圈：陌生人（冷启动）**
- 冷邮件、冷消息——这有效。Gumroad 就是这么做大的。
- Sahil 在网上搜索所有可能受益的人，逐一发邮件，发了几千次。
- 每次被拒绝都是一次学习机会。

### 销售不是脏话

- 你不是在说服任何人。你是在帮助他们。
- 把每次失败的转化变成洞察——要么找错人了，要么产品需要改进。
- 销售是教育过程：客户了解你，你了解什么有效。

### 关键数据

```
手动销售 = 早期 99% 的增长
口碑传播 = 后期 99% 的增长

你需要的客户比你想的少得多：
Slack IPO 时：575 个客户 = 40% 的收入。

如果产品 ¥69/月，你需要 200 个客户 = ¥13,800/月。
每个工作日签 1 个客户，不到一年就到了。
```

### 何时"发布"

**有了 100 个付费客户之后再发布。** 发布应该是庆祝社区支持的派对，不是获客策略。

## 7. 定价策略（Pricing）

### 核心原则

**永远收费。** 免费和 ¥1 之间有巨大的差距。行为经济学家 Dan Ariely 称之为"零价格效应"——人们会排队领免费布朗尼，但你收 1 分钱，队伍就消失了。不收费，你既无法生存，也无法了解客户真正看重什么。

### 两种定价模型

| 模型 | 做法 | 适用场景 |
|------|------|---------|
| **成本加成** | 成本 + 利润率（20-50%） | 实物产品、成本清晰的服务 |
| **价值定价** | 基于客户获得的价值，与成本无关 | 软件、数字产品、高感知价值服务 |

### 定价原则

1. **从低开始，逐步提价。** 产品改进了，价格上涨是正常且健康的。
2. **定价不是永久的。** 跟其他一切一样，是迭代的。
3. **分层定价是目标。** 像机票——经济舱、商务舱、头等舱。同一目的地，不同体验。
4. **零价格效应。** 永远不要把免费作为默认选项。
5. **免费试用是标配。** 但必须有清晰的付费转化路径。

### 财务独立的数学

```
你每月需要多少钱维持生活？→ ¥X
你的产品定价是多少？→ ¥Y/月
你需要多少客户？→ X / Y = N 个客户
每个工作日签 1 个客户（每年 260 个），多久达到？→ N / 260 年

例：需要 ¥15,000/月，产品 ¥99/月
→ 需要 152 个客户
→ 每天签 1 个 ≈ 7.5 个月
```

## 8. 营销计划（Marketing Plan）

### 核心原则

**营销是规模化的销售。** 但别把营销和广告搞混。营销是交朋友，不是买头条。先花时间，再花钱。

### 前置条件

在做营销之前，你应该已经有：
- 你所属的社区
- 人们在为之付费的产品
- ~100 个客户（复购客户 = 产品市场匹配）
- 一对一销售的经验

### 三层内容策略

**Level 1：教育** — 分享你从建设中学到的东西。如果你有 100 个客户，你已经学到了 100 件值得分享的事。

**Level 2：激励** — 超越教学，进入激励。分享你的旅程、挣扎和成功。像 Gimlet 的 *StartUp* 播客一样真诚。

**Level 3：娱乐** — 最难但传播最广。"创业：每周工作 60 小时，这样你就不用每周工作 40 小时了。"

### 拥有你的受众

```
社交媒体 = 租来的地（算法可以变，账号可以被封）
邮件列表 = 自己的地（直达客户，没有算法）

从第一天就开始建邮件列表。
用有价值的东西换邮件（指南、PDF、清单）。
每个邮件订阅者比社交媒体关注者值钱得多。
```

### 花钱是最后一步

- 你看到的大部分增长都是花钱买的。别被误导。
- 只有在有机增长已经跑通之后才花钱做广告。
- 先花钱在客户身上（奖励、忠诚度），再花钱获客。
- 常识规则：获客成本不能超过客户终身价值。

## 9. 可持续增长（Grow Sustainably）

### 核心原则

**盈利是超能力。** 它给你无限跑道、清晰度和控制权。花得比赚的少。听起来简单，做起来不容易。当你盈利了，你可以慢慢来，做对的决定，按自己的节奏走。

### 成本控制规则

1. **给自己发尽可能少的工资。** Sahil 在旧金山拿 $36K/年起步。2015 年困难时，给自己发 $0。
2. **雇软件，不雇人。** 用 Pilot 做会计、用 Zapier 做自动化。软件便宜，人贵。
3. **别租办公室。** 远程是默认。办公室带来大量关联成本。
4. **先外包。** 你 + AI 机器人大军 → 然后自由职业者 → 最后才是全职员工。
5. **等到痛了再招人。** 这意味着你已经有一个成熟的业务让新人融入。

### 增长心态

- 你不需要统治市场、颠覆什么、征服竞争对手
- 世界上最长寿的企业是一些最小的：餐厅、酒店、家族公司
- 工作更多小时不一定意味着更快增长
- 你的公司会以客户决定的速度增长

### 盈利的力量

```
当你盈利时：
- 跑道是无限的。除非你做蠢事，否则不会死。
- 你可以慢慢发、深思熟虑地做产品。
- 别人可能在亮眼指标上火箭般超过你，但他们 10 年后不会在了。你会。
```

### 融资（如果必须的话）

- **先自力更生。** 盈利在任何融资谈话中都给你筹码。
- **考虑众筹。** 把客户变成投资人，利益绑定。
- **新型 VC 替代**：Earnest Capital、Indie.vc、Tinyseed——投资可持续企业的基金。
- **如果拿 VC**：盈利意味着更低的稀释和更多的控制权。

## 10. 公司价值观（Company Values）

### 核心原则

**先建文化，再招人。** 价值观不是通用的两字标语——是用非显而易见的方式，表达非显而易见的东西。

### 如何创建价值观

1. **你相信什么是大多数人不相信的？** 价值观应该是非显而易见的，甚至有些极化的。
2. **没人监督时，人们应该怎么表现？** 价值观是为没有经理在场的时刻准备的。
3. **即使一个人业绩出色，你也会因为什么而开除他？** 这揭示了你的真正价值观。
4. **即使没有直接帮助底线，你也会庆祝什么？** 这也是价值观。
5. **用故事写，不是口号。** "以用户为中心"是口号。Nordstrom 在服装店接受退轮胎，是通过故事传达的价值观。

### Gumroad 的价值观（参考）

| 价值观 | 含义 |
|--------|------|
| 以作品论英雄 | 重要的是创作者和客户的体验，不是谁来得早谁加班多 |
| 追求超线性回报 | 固定时间，无限价值。每天都在产出复利回报 |
| 人人都是 CEO | 你是你职能的 CEO，不是等指令的员工 |
| 敢于透明 | 完全信息对称。公开薪资、公开财务、没有秘密 |

## 11. 极简主义决策检查（Minimalist Review）

对任何商业决策，用这个框架过一遍：

| 检查项 | 你的答案 |
|--------|---------|
| 这服务于我的社区/客户吗？ | |
| 这是最简单的方法吗？ | |
| 这改善盈利能力吗？ | |
| 如果不行，能回退吗？（避免不可逆决策） | |
| 我在花时间还是花钱？（先花时间） | |
| 客户要求了这个吗？ | |
| 这符合我的价值观吗？ | |
| 一年后我还想要这个吗？ | |

### 极简创业金句速查

- "别发布。先卖给你的前 100 个客户。"
- "等到痛了再招人。"
- "你的失败会消退，你的成功会留下来并复利增长。"
- "盈利让你脱离电网。然后你带着无限跑道用心增长。"
- "自私地为自己建造对的企业，同时无私地服务他人的社区。"

### 与 Vibe Coding 的终极结合

```
极简主义 + Vibe Coding 的黄金流程：

1. 社区中发现问题 → 不写代码
2. 手动为 3 个人解决 → 不写代码
3. 写魔法纸条（流程文档）→ 不写代码
4. 卖给 10 个人 → 不写代码
5. 验证通过 → 用 AI 写代码，一个周末产品化
6. 卖到 100 个人 → 根据反馈用 AI 迭代
7. 开始营销 → 教育/激励/娱乐三层内容
8. 保持盈利 → 你 + AI + 外包 = 极简团队
9. 定义文化 → 准备扩展时的基础
10. 每个决策过极简检查 → 持续复利增长
```

## 12. 方法论谱系与背书（Methodology & Endorsements）

本技能的方法论不是凭空发明的，而是站在这些巨人的肩膀上：

### 精益创业（Lean Startup）—— Eric Ries

Eric Ries 在 2011 年出版《The Lean Startup》，彻底改变了创业方法论。核心框架：

```
构建（Build）→ 度量（Measure）→ 学习（Learn）→ 循环

关键概念：
- MVP（最小可行产品）：用最小成本测试核心假设
- 验证式学习（Validated Learning）：每次迭代必须产生可度量的学习
- 转型（Pivot）：假设被推翻时，勇敢调整方向
- 创新核算（Innovation Accounting）：用数据而非直觉衡量进展
```

**与极简创业的关系**：Sahil Lavingia 的方法是精益创业在个人/小团队场景下的极致演绎——把 MVP 缩小到"手动交付"，把验证缩短到"这周就收到钱"。

**与 Vibe Coding 的关系**：AI 让 Build-Measure-Learn 循环从"周"级缩短到"天"级。但如果你跳过 Measure 和 Learn 直接 Build，AI 只是帮你更快地做出没人要的东西。

### 客户开发（Customer Development）—— Steve Blank

Steve Blank 是精益创业的奠基者，Eric Ries 的导师。《The Four Steps to the Epiphany》提出：

```
客户发现（Discovery）→ 客户验证（Validation）→ 客户创建（Creation）→ 公司建设（Building）

核心洞察：
- "走出办公室"（Get out of the building）—— 你的假设在办公室里永远不会被推翻
- 产品/市场契合（Product-Market Fit）之前，不要规模化
- 创始人必须亲自做销售 —— 不是因为省钱，而是因为要学习
```

### Indie Hacker 代表人物

**Pieter Levels (@levelsio)**
- 一个人运营 Nomad List、Remote OK 等多个年收入百万美元的产品
- 极致的"一人公司"模型：不融资、不招人、不用框架
- 名言：用最无聊的技术栈，做最赚钱的产品
- 他用 PHP + jQuery 打败了无数用 React + microservices 的团队

**Sahil Lavingia (@shl)**
- Gumroad 创始人，本技能核心方法论来源
- 从硅谷 VC 模式转型为盈利优先的极简创业
- Gumroad 2024 年收入超 $20M，团队不到 30 人
- 名言：先手动，再自动化；先盈利，再增长

**DHH (David Heinemeier Hansson)**
- Basecamp/37signals 联合创始人，Ruby on Rails 创造者
- 《Rework》《It Doesn't Have to Be Crazy at Work》作者
- 20+ 年持续盈利，拒绝 VC，证明"小而美"可以长期存活
- 名言：选择做一个盈利的小公司，而不是一个亏损的大公司

**Patrick McKenzie (@paborin)**
- 从独立开发者到 Stripe 顾问
- SaaS 定价和 B2B 营销的教父级人物
- 名言：收费要大胆，你的产品几乎肯定定价过低

### Paul Graham (YC) 的创业智慧

Y Combinator 创始人，投资了 Airbnb、Stripe、Dropbox 等。他的文章是创业者圣经：

```
核心原则：
- "Do things that don't scale" —— 先手动，后自动化（和极简创业完美一致）
- "Make something people want" —— YC 的唯一 motto
- "Launch early" —— 你的第一版应该让你尴尬，如果不尴尬就是发布太晚了
- "Talk to users" —— 创业初期最重要的两件事：写代码 + 跟用户聊
```

### 精益方法论速查表

| 方法论 | 创始人 | 核心一句话 | 适用阶段 |
|--------|--------|-----------|---------|
| **精益创业** | Eric Ries | Build → Measure → Learn，快速迭代 | 0→1 验证期 |
| **客户开发** | Steve Blank | 走出办公室，先找到市场再做产品 | 想法验证期 |
| **极简创业** | Sahil Lavingia | 先手动交付，再自动化；先盈利，再增长 | 0→1 全过程 |
| **Rework** | DHH & Jason Fried | 做得更少，但做得更好；拒绝增长崇拜 | 持续运营期 |
| **YC 方法论** | Paul Graham | 做人们想要的东西，做不可规模化的事 | 早期获客期 |
| **Indie Hacker** | Pieter Levels 等 | 一人公司、无聊技术栈、利润至上 | 全过程 |

### 这些方法论的共识

尽管表达方式不同，这些创业大师有惊人的共识：

```
1. 先验证，后建造 —— 所有人都同意
2. 跟用户直接对话 —— 所有人都同意
3. 尽早收费 —— 所有人都同意
4. 做得少但做得对 —— 所有人都同意
5. 盈利比增长重要 —— 除了 VC 之外所有人都同意

在 Vibe Coding 时代，这些共识更加重要——
因为 AI 让"建造"变得太容易了，
反而让"验证"和"选对方向"成了唯一的竞争壁垒。
```
