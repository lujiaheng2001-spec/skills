## 1. 商业化时机判断

### 什么时候开始收费？

**答案：尽可能早。**

很多 PM 犯的错误是"等产品完善了再收费"。但在 vibe coding 时代，你不缺产品（3 天就能做一个），你缺的是验证付费意愿的信号。

```
时机判断决策树：

这个产品解决的问题，用户目前怎么解决？
│
├─ 花钱买别的工具 → 立即收费（他们已经有付费习惯）
│
├─ 花时间手动做 → 可以收费（你帮他们省时间 = 有价值）
│   └─ 但要明确：省下的时间值多少钱？
│
├─ 忍着不解决 → 谨慎（可能痛点不够大）
│   └─ 先用免费版验证需求是否真实
│
└─ 有免费替代方案 → 你的差异化价值是什么？
    └─ 如果说不清楚，先找到差异化再谈收费
```

### 收费前的验证清单

- [ ] 至少 5 个人（非亲友）愿意为这个东西付费
- [ ] 你能用一句话说清楚"为什么值这个价"
- [ ] 有至少一个用户在没有提示的情况下主动问"这个多少钱？"
- [ ] 竞品分析完成，你知道用户的价格预期区间

## 2. 变现模型深度解析

### 2.1 订阅制（Subscription）—— 最适合 Vibe Coding 产品的模型

**为什么是首选**：
- 收入可预测（MRR），投资人最爱看这个
- 天然的复利结构——只要留存好，收入自动增长
- Stripe + Next.js 集成已经非常成熟，AI 生成的代码质量也不错

**典型结构**：

```
Free    → 核心功能受限（如：3 个项目、100 条记录）
Pro     → 全部功能（如：无限项目、API 访问、优先支持）
Team    → Pro + 协作功能（如：多成员、角色权限、共享工作区）

关键：Free 到 Pro 的升级触发点必须设计在用户已经产生"沉没成本"之后
      用户已经创建了数据、养成了习惯、形成了依赖——这时候收费阻力最小
```

**实现要点**：
- 从 Day 1 集成 Stripe（不要等"以后再加"）
- 用 Feature Flag 控制功能开关（而不是做两套代码）
- Webhook 处理：订阅创建、续费成功、续费失败、取消、退款——每一个都要处理
- 宽限期（Grace Period）：付费失败后给 3-7 天宽限，不要立即锁定功能

### 2.2 用量付费（Usage-Based）—— AI 产品的最佳搭档

**为什么适合 AI 产品**：
AI 功能（调用 LLM API、图片生成）有真实的边际成本。让重度用户多付，轻度用户少付，双方都满意。

**典型结构**：

```
免费额度  → 每月 100 次 AI 查询
按量计费  → 超出后 $0.01/次
月度包    → $19/月 = 5,000 次（比按次便宜 60%）

或者 Credit 模式：
$10 = 1,000 credits
不同操作消耗不同 credits（简单查询 1 credit，复杂生成 10 credits）
```

**实现要点**：
- 需要一个计量系统（每次 API 调用都要记录和计数）
- 实时余额展示（用户需要知道自己还剩多少）
- 接近额度时提醒（80% 和 100% 时各发一次通知）
- 超额处理策略（阻断 vs 继续使用后结算——选择取决于你的信任模型）

### 2.3 免费增值（Freemium）—— 网络效应产品的标配

**核心逻辑**：免费用户是获客手段，不是慈善。他们的价值在于创造内容、传播产品、形成网络效应。

**付费墙设计原则**：

```
✅ 好的付费墙（用户觉得合理）：
   - 功能性限制：免费 3 个项目，付费无限
   - 容量限制：免费 100MB 存储，付费 10GB
   - 高级功能：AI 分析、自动化、团队协作

❌ 差的付费墙（用户觉得恶心）：
   - 限制核心体验：能看到数据但不能导出
   - 人为制造不便：免费版必须看 10 秒广告才能继续
   - 功能残缺：核心功能缺一半，免费版几乎没法用
```

### 2.4 一次性付费 —— 最简单但没有复利

适用于：模板、工具包、一次性服务、数字内容。

**没有复利怎么办？** 做产品矩阵：

```
产品 A（$29 模板）→ 每月卖 50 份 = $1,450/月
产品 B（$49 工具）→ 每月卖 30 份 = $1,470/月
产品 C（$99 课程）→ 每月卖 20 份 = $1,980/月
                    合计 = $4,900/月

每做一个新产品，总收入就在前一个基础上叠加
这不是单产品的复利，而是产品组合的复利
```

## 3. 定价策略实操

### 定价的三种方法

| 方法 | 做法 | 适用场景 |
|------|------|---------|
| **竞品参照法** | 查看 3-5 个竞品的定价，在同一区间内定位 | 市场已有成熟竞品 |
| **价值倍数法** | 你帮用户省了 X 元/小时，定价 = X × 节省小时数 × 10-30% | 效率类工具 |
| **用户访谈法** | 直接问 20 个潜在用户"你愿意为这个付多少钱" | 创新品类 |

### Van Westendorp 价格敏感度模型（快速版）

问潜在用户四个问题：
1. 什么价格你觉得**太便宜**，会怀疑质量？
2. 什么价格你觉得**便宜**，是个好交易？
3. 什么价格你觉得**开始贵了**，但还是会考虑？
4. 什么价格你觉得**太贵**，绝对不会买？

四个答案画成图，交叉点就是最优价格区间。20 个人的样本就够得到有用的信号。

### 定价实验操作指南

```
Step 1：确定初始价格（竞品参照 + 直觉）
Step 2：上线 A 组价格（当前价 100%）和 B 组价格（当前价 120%）
Step 3：跑 2 周，比较转化率
Step 4：如果 B 组转化率没有显著下降（<15% 降幅），把价格提到 120%
Step 5：重复，直到找到转化率开始明显下降的拐点

⚠️ 注意：对已付费用户保持原价（Grandfather clause）
        涨价只对新用户生效，否则流失率会飙升
```

## 4. 复利增长引擎

### 4.1 留存是一切的基础

```
留存 vs 获客的数学：

假设你每月获取 100 个新用户，月留存率分别是 95% 和 80%：

         95% 留存    80% 留存
第 1 月    100          100
第 6 月    477          335
第 12 月   874          503
第 24 月  1,527         500（稳态）

差距不是线性的，而是指数级的。这就是复利。
```

**提升留存率的杠杆（按 ROI 排序）**：

| 杠杆 | 预期效果 | 实现成本 |
|------|---------|---------|
| **新用户引导（Onboarding）优化** | 留存率提升 10-25% | 低（改 UI 流程） |
| **建立使用习惯** | 留存率提升 5-15% | 中（触发式通知、提醒） |
| **数据锁定（Data lock-in）** | 大幅降低流失率 | 低（用户创造的数据越多越不舍得走） |
| **社交粘性** | 留存率提升 10-20% | 中高（需要多人协作功能） |
| **持续交付新价值** | 稳定留存 | 持续投入 |

### 4.2 扩展收入：让老客户花更多钱

NDR > 100% 的秘诀不是涨价，而是让用户自然地花更多钱：

```
扩展收入的五种路径：

1. 用量增长  → 用户的数据/调用量自然增长，自动升级到更高价位
2. 席位扩展  → 用户邀请团队成员加入
3. 功能升级  → 用户需要高级功能，从 Pro 升级到 Team/Enterprise
4. 附加服务  → 提供增值服务（定制化、培训、优先支持）
5. 交叉销售  → 向现有用户推荐你的其他产品
```

### 4.3 降低 CAC：让获客成本趋近于零

Vibe coding 产品最大的优势——你可以快速做出新功能来吸引用户，而不是砸钱投广告。

**零成本获客渠道（按优先级）**：

| 渠道 | 做法 | 时间投入 | 效果周期 |
|------|------|---------|---------|
| **SEO + 内容** | 写解决用户痛点的文章，自然排名引流 | 高初始、后续递减 | 3-6 个月见效，长期复利 |
| **产品内传播** | "Powered by XX"水印、邀请功能、分享链接 | 一次性开发 | 立即生效 |
| **社区运营** | 在目标用户出没的社区提供价值 | 持续但每天 30 分钟 | 1-3 个月 |
| **Product Hunt 上线** | 精心准备一次产品发布 | 集中 1 周 | 一次性流量 + 长尾 SEO |
| **开源引流** | 把非核心部分开源，吸引开发者 | 视规模 | 长期 |

## 5. 关键指标体系与监控

### PM 的商业化仪表盘

这些指标应该在上线第一天就开始追踪，而不是"等有规模了再说"：

```
┌─────────────────────────────────────────────────┐
│              商业化核心仪表盘                      │
├───────────────┬─────────────────────────────────┤
│ 收入指标       │ MRR / ARR / MRR 增长率           │
│               │ 新增 MRR / 扩展 MRR / 流失 MRR    │
│               │ NDR（净收入留存率）                 │
├───────────────┼─────────────────────────────────┤
│ 用户指标       │ 注册用户 / 付费用户 / 转化率       │
│               │ DAU / WAU / MAU                   │
│               │ 活跃用户中付费占比                  │
├───────────────┼─────────────────────────────────┤
│ 健康指标       │ LTV / CAC / LTV:CAC ratio        │
│               │ 回本周期（Payback Period）          │
│               │ 月度流失率（Churn Rate）            │
│               │ ARPU（每用户平均收入）              │
├───────────────┼─────────────────────────────────┤
│ 行为指标       │ 用户激活率（注册→核心行为）         │
│               │ 功能使用率（哪些功能驱动留存）       │
│               │ 升级触发事件（什么行为后用户升级）    │
└───────────────┴─────────────────────────────────┘
```

### 各指标的健康标准

| 指标 | 危险区 | 健康区 | 优秀区 |
|------|--------|--------|--------|
| 月度用户流失率 | > 8% | 3-5% | < 2% |
| LTV:CAC | < 2:1 | 3:1 - 5:1 | > 5:1 |
| 回本周期 | > 18 个月 | 8-12 个月 | < 6 个月 |
| NDR | < 90% | 100-115% | > 120% |
| 免费→付费转化率 | < 1% | 2-5% | > 7% |
| MRR 月增长率 | < 5% | 10-15% | > 20% |

### 预警信号

当以下情况出现时，停下来审视商业模型：

| 信号 | 可能的问题 | 行动 |
|------|----------|------|
| MRR 连续 3 个月不增长 | 获客渠道饱和 or 产品市场不匹配 | 做用户访谈、测试新渠道 |
| 流失率突然上升 | 产品问题 or 竞品出现 or 价格敏感 | 分析流失用户、做挽留流程 |
| CAC 持续上升 | 广告效率下降 or 市场拥挤 | 转向有机获客渠道 |
| 付费转化率下降 | 免费方案太慷慨 or 付费价值不清晰 | 调整 paywall 位置 |
| ARPU 下降 | 用户降级多于升级 | 审视定价结构、增加高价值功能 |

## 6. 从零到稳定复利的阶段路线图

### Phase 0：验证（第 1-4 周）

```
目标：证明有人愿意付费
做法：
  - Vibe code 一个最小可用产品
  - 找到 10 个目标用户做测试
  - 至少 3 个人表示愿意付费
  - 确定初始定价
产出：付费意愿验证完成，初始价格确定
```

### Phase 1：第一笔收入（第 1-3 月）

```
目标：MRR > $0 → 第一个付费用户
做法：
  - 集成 Stripe
  - 发布到 Product Hunt / 目标社区
  - 手动获客（DM、社区发帖、朋友推荐）
  - 每周跟踪：注册数、付费数、流失数
产出：$100-500 MRR
关键：这个阶段不要优化，要验证
```

### Phase 2：产品市场匹配（第 3-6 月）

```
目标：月度增长 > 10%、流失率 < 8%
做法：
  - 根据用户反馈迭代产品
  - 优化 onboarding 流程
  - 建立内容 / SEO 获客渠道
  - 开始追踪完整的指标仪表盘
产出：$500-3,000 MRR
关键：如果增长停滞，可能是 PMF（产品市场匹配）没找到，
      需要回去调整产品而不是加大营销
```

### Phase 3：增长引擎（第 6-12 月）

```
目标：可预测、可重复的增长
做法：
  - 建立 2-3 个稳定的获客渠道
  - 优化付费转化漏斗
  - 设计扩展收入路径（高级方案 / 团队方案）
  - 开始关注 NDR
产出：$3,000-10,000 MRR
关键：找到你的 "北极星指标"——
      那一个最能预测用户长期留存和付费的行为指标
```

### Phase 4：稳定复利（第 12 月+）

```
目标：NDR > 100%，增长自驱
做法：
  - NDR 优化（减少流失 + 增加扩展收入）
  - 自动化运营（onboarding、邮件、续费提醒）
  - 产品矩阵扩展（做第二个产品 or 进入相邻市场）
  - 考虑是否需要融资或保持 bootstrapped
产出：$10,000+ MRR，且月环比增长稳定
关键：这时候你赚的钱开始"利滚利"了——
      老用户的扩展收入 + 新用户的付费 = 复合增长
```

## 7. Vibe Coding 产品的成本结构

### 你需要知道的成本真相

Vibe coding 做产品的好处是开发成本几乎为零，但运营成本不是零。

```
月度运营成本估算（典型 SaaS）：

基础设施：
  - Vercel Pro          $20/月
  - Supabase Pro        $25/月
  - 域名               $1/月（年付 $12）
  - SSL                $0（Let's Encrypt）

SaaS 工具：
  - Stripe 手续费       2.9% + $0.30/笔
  - Sentry（错误监控）   $0-26/月
  - 邮件服务（Resend）   $0-20/月

AI 功能成本（如有）：
  - OpenAI API          按用量（$0.002-0.06/千 token）
  - 向量数据库           $0-25/月

总计：$50-150/月（用户量小时）
```

**定价时要考虑的毛利率**：

```
毛利率 = (收入 - 直接成本) / 收入 × 100%

SaaS 健康毛利率：70-85%

如果你的产品 $29/月，每个用户的直接成本（服务器 + API 调用）不超过 $5-8
毛利率 = ($29 - $7) / $29 = 75% ✅

但如果你的 AI 功能每用户每月消耗 $15 的 API 费用...
毛利率 = ($29 - $15) / $29 = 48% ❌
→ 需要涨价，或限制免费额度，或优化 API 调用效率
```

## 附录：快速参考

### 支付集成检查清单（Stripe）

- [ ] Stripe API key 在环境变量中（绝不硬编码）
- [ ] 使用 Stripe Checkout 或 Payment Element（不要自己做支付表单）
- [ ] Webhook 端点已配置并验证签名
- [ ] 处理了这些事件：checkout.session.completed, invoice.paid, invoice.payment_failed, customer.subscription.deleted
- [ ] 测试模式和生产模式使用不同的 API key
- [ ] 有 Webhook 处理失败时的重试机制
- [ ] 价格变更不影响现有订阅用户（Grandfather clause）
- [ ] 用户可以在产品内管理订阅（升级/降级/取消）

### 定价心理学速查

| 锚定效应 | 先展示最贵的方案 | 让中间方案显得合理 |
| 价格尾数 | $29 而不是 $30 | 感知上更便宜 |
| 年付折扣 | 年付 = 月付 × 10（省 2 个月） | 提前锁定收入、降低流失 |
| 默认选中 | 推荐方案用高亮 + "Most Popular" | 减少决策负担 |
| 功能对比表 | 让用户一眼看到差异 | 让升级理由可视化 |
| 试用期 | 14 天免费试用（需绑定信用卡） | 体验后转化率更高 |
